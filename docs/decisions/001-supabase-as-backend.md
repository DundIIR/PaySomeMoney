# 001. Supabase как бэкенд вместо своего сервера

**Статус:** принято
**Дата:** 2026-06-11

## Контекст

Проекту нужен бэкенд для трёх операций: создать платёж в ЮKassa, принять вебхук, отдать статистику по токену. Плюс база для хранения платежей. Есть VPS Timeweb, на котором можно было бы поднять свой сервер (Node/whatever) + Postgres.

## Решение

Используем Supabase: managed Postgres + Edge Functions. На VPS едет **только статика фронта** (nginx).

## Обоснование

- Не нужно администрировать БД и рантайм бэкенда на VPS — меньше точек отказа и работы по поддержке.
- Edge Functions покрывают все три серверные операции; секреты ЮKassa хранятся в Supabase Secrets.
- RLS позволяет полностью закрыть таблицу от anon-ключа без написания собственного слоя авторизации.
- Supabase Auth при этом не используем — для проекта с одной кнопкой полноценная авторизация избыточна, доступ контролируется через access_token (см. [002](002-access-token-scheme.md)).

## Последствия

- Бэкенд-логика пишется на Deno (рантайм Edge Functions).
- Зависимость от внешнего сервиса: если Supabase лежит — не работают платежи и статистика, но сам сайт (статика) остаётся доступен.
- Проект создан в регионе eu-central-1.
