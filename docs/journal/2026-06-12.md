## Сделано

- Написана Edge Function `create-payment` ([supabase/functions/create-payment/index.ts](../../supabase/functions/create-payment/index.ts)):
  - генерит `access_token` через `crypto.randomUUID()`;
  - создаёт запись `pending` в `payments` через service role;
  - создаёт платёж в ЮKassa (55.00 RUB, capture: true, токен в metadata, Idempotence-Key = access_token);
  - при ошибке ЮKassa удаляет запись и возвращает 500;
  - CORS открыт для любого origin (временно);
  - возвращает `{ access_token, confirmation_url }`.
- Задеплоена в проект `tpnrzwstakixiaeuwlop` через MCP (версия 1, ACTIVE, verify_jwt включён).
- Написана Edge Function `payment-webhook` ([supabase/functions/payment-webhook/index.ts](../../supabase/functions/payment-webhook/index.ts)):
  - обрабатывает только событие `payment.succeeded`, остальные игнорирует с 200;
  - находит платёж по `yookassa_payment_id`, ставит `status = succeeded` и `paid_at = now()`;
  - всегда возвращает 200 (даже если запись не найдена) — чтобы ЮKassa не ретраила; ошибки логируются через `console.error`;
  - CORS не нужен (вебхук идёт сервер-сервер).
- Задеплоена через MCP (версия 1, ACTIVE, **verify_jwt отключён** — ЮKassa не знает про наш JWT).
- Написана Edge Function `get-stats` ([supabase/functions/get-stats/index.ts](../../supabase/functions/get-stats/index.ts)):
  - GET с query-параметром `token`; без токена — 400;
  - доступ: запись с `access_token = token` и `status = succeeded`, `paid_at` не старше 24 часов; иначе 403;
  - агрегаты считаются в JS по выборке всех succeeded-записей (записей мало, это ок);
  - ответ: `{ total_count, total_amount, daily: [{ date, count, amount }] }`, суммы в рублях (БД хранит копейки), `daily` отсортирован по дате;
  - CORS открыт, как в `create-payment`.
- Задеплоена через MCP (версия 1, ACTIVE, verify_jwt отключён — защита через наш access_token).
